Observations Pro

Privacy Policy

Last updated: June 16, 2026

Observations Pro 1.0 is designed as a standalone, device-local behavior observation app. The first version does not require an account, school sync, a web dashboard, or a production API connection.

Data Stored On Device

The app stores observation workspace data locally on the device, including:

  • Student names.
  • Optional grade, room, and school labels.
  • Behavior target names and recording types.
  • Active observation drafts used to resume interrupted sessions.
  • Completed observation summaries and local report data.
  • User preferences such as haptic feedback, interval length, and session duration.
  • Local onboarding and workspace settings.

Data Shared By User Action

Users can choose to share a session summary through the native iOS share sheet. When a user chooses to share, the destination and handling of that summary are controlled by the selected iOS sharing option.

Data Not Collected By The Developer

For version 1.0, the developer does not collect user-entered observation data, student information, analytics events, advertising identifiers, location, photos, videos, audio recordings, contacts, calendar data, or account credentials.

Permissions

Version 1.0 does not request microphone, camera, location, photo library, contacts, calendar, notification, tracking, or Bluetooth permissions.

Future Changes

If a future version adds optional cloud sync or account features, this policy should be updated before that version is submitted.
